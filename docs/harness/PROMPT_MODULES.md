# PROMPT_MODULES.md

# 可复用 Prompt 模块库

这份文件不直接执行任务，而是定义以后可以复用、拼装、注入的 prompt 模块。

目标：把经验话术变成稳定模块，避免每次临时重写。

---

## 1. BOOTSTRAP_MODULE

### 用途
用于多步骤任务开始前，强制补齐最小上下文。

### 注入内容
- 当前任务类型：调研 / 文档 / 代码 / 工作流
- 当前工作目录
- 最终输出位置
- 可用工具与优先工具
- 完成标准
- 验证方式
- 是否需要 `TODOS.md`

### 使用场景
- 所有多步骤任务
- cron 生成类任务
- 文档落地类任务
- 代码修改类任务

---

## 2. TASK_ROUTER_MODULE

### 用途
用于在任务开始时快速决定走哪套模板。

### 路由问题
1. 这是单步还是多步骤任务？
2. 主要属于哪一类？
3. 是否要求落飞书？
4. 是否要求验证证据？
5. 是否需要子任务拆分？

### 输出结果
- 主模板类型
- 是否 bootstrap
- 是否写 `TODOS.md`
- 是否需要文档落地规则

---

## 3. DOC_WORKFLOW_MODULE

### 用途
把文档类任务的动作顺序固定下来。

### 固定顺序
1. 确定 folder token / doc token
2. 确定新建还是更新
3. 写入正文
4. 校验写入结果
5. 回链路

### 约束
- 没目标位置，不开始
- 没写入校验，不算完成
- 没回链，不算完成

---

## 4. CHECKLIST_MODULE

### 用途
用于任务输出前做统一检查。

### 核心检查项
- 需求是否完成
- 是否遗漏显式要求
- 是否有验证证据
- 是否落到指定位置
- 是否存在失败残留

### 说明
这是一层通用检查，具体 checklist 细节仍以 `HARNESS_SPEC.md` 为准。

---

## 5. FAILSAFE_MODULE

### 用途
用于减少重复失败、垃圾产物、错误收尾。

### 规则
- 失败文档必须清理
- 不允许保留空标题文档
- 同类动作最多重试一次
- 第二次失败必须停止并汇报
- 不允许无限重试

### 适用场景
- 飞书文档写入
- cron 自动生成任务
- 任何会产生外部产物的任务

---

## 6. OUTPUT_MODULE

### 用途
统一规范最终输出内容。

### 成功输出
只保留：
- 做了什么
- 结果是什么
- 证据是什么
- 链接 / token / 路径是什么

### 失败输出
只保留：
- 失败在哪一步
- 已清理什么残留
- 是否停止重试
- 下一步建议
