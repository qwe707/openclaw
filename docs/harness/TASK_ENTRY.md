# TASK_ENTRY.md

# 任务入口分类规则

这份文件定义：任务一进入系统，先怎么分类，再决定套哪套模板。

不是执行细节，而是任务入口决策规则。

---

## 一、入口判断顺序

每个任务进入后，按这个顺序判断：

1. 是否为多步骤任务
2. 主要任务类型是什么
3. 是否需要文档落地
4. 是否需要验证证据
5. 是否需要建立 `TODOS.md`
6. 是否需要进入完成闸门

---

## 二、任务类型分类

### 1. 调研类
触发特征：
- 查资料
- 查官方文档
- 对比方案
- 提炼结论

默认要求：
- 用调研模板
- 要区分事实与判断
- 输出前必须有结构化结论

### 2. 文档类
触发特征：
- 写 doc
- 更新 doc
- 写总结/报告/说明
- 要求落飞书

默认要求：
- 用文档模板
- 必须套 `DOC_WORKFLOW.md`
- 必须回链路

### 3. 代码类
触发特征：
- 改文件
- 修 bug
- 配置修改
- 测试 / 构建 / 验证

默认要求：
- 用代码模板
- 必须建立 `TODOS.md`
- 必须说明验证结果

### 4. 工作流类
触发特征：
- cron
- 自动化流程
- 多步骤交付
- 子任务编排

默认要求：
- 用工作流模板
- 默认视为多步骤任务
- 必须有产物落点与回查路径

---

## 三、Bootstrap 判定

满足以下任一条件，必须 bootstrap：
- 多步骤任务
- 要落飞书
- 要改代码或配置
- 要跑 cron / 自动流程
- 要生成最终可交付产物

---

## 四、TODOS 判定

满足以下任一条件，必须建立 `TODOS.md`：
- 任务超过 2 步
- 涉及多个文件 / 多个工具
- 需要验证与收尾
- 需要分阶段汇报

---

## 五、文档落地判定

满足以下任一条件，必须进入文档落地规则：
- 用户要求写飞书
- 任务结果是报告/总结/学习文档
- cron 任务需要沉淀文档产物

此时必须套用：
- `TASK_TEMPLATE.md` 的文档模板
- `DOC_WORKFLOW.md`

---

## 六、输出前判定

所有非单句即时回复任务，在输出前都必须进入：
- `COMPLETION_GATE.md`

如果过不了完成闸门，就不能按“已完成”回复。
