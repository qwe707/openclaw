# TASK_TEMPLATE.md

# 固定任务模板库

这份文件用于把常见任务固定成可复用模板。

不是原则说明，而是执行模板。

---

## 1. 调研类任务模板

### 任务识别
适用于：
- 查资料
- 查官方文档
- 比较方案
- 输出结构化结论

### 启动 Bootstrap
开始前必须确认：
- 用户真正的问题是什么
- 需要原文、官方文档，还是二手总结即可
- 最终输出到聊天、飞书文档，还是两者都要
- 使用哪种搜索路径与来源优先级

### 执行步骤
1. 明确问题边界
2. 找核心来源 / 原文
3. 区分事实、判断、建议
4. 提炼结构化结论
5. 如果用户要求沉淀，则写入指定位置

### 完成 Checklist
回复前必须检查：
- 是否读了核心来源
- 是否明确区分事实与判断
- 是否直接回答了用户问题
- 是否说明“这意味着什么”
- 若要求存档，是否已落到指定位置
- 是否有可引用证据

---

## 2. 文档类任务模板

### 任务识别
适用于：
- 新建文档
- 更新文档
- 输出总结/报告/说明
- 飞书 Doc / Drive 落地

### 启动 Bootstrap
开始前必须确认：
- 写到哪个 doc / folder
- 是新建还是更新
- 输出形式是长文、精简版还是结构化摘要
- 最终需要返回什么链接或回链信息

### 执行步骤
1. 确定目标 folder / doc
2. 确定新建或更新策略
3. 写入内容
4. 校验写入结果
5. 把最终链接 / 结果返回给用户

### 完成 Checklist
回复前必须检查：
- 是否写到了正确 doc / folder
- 文档标题 / 命名是否正确
- 内容是否完整
- 写入是否成功
- 是否返回最终链接 / 路径
- 是否有写入成功的证据

---

## 3. 代码类任务模板

### 任务识别
适用于：
- 改代码
- 修 bug
- 重构
- 配置改造
- 测试 / 构建 / 验证

### 启动 Bootstrap
开始前必须确认：
- 项目根目录
- 目标文件或模块
- 验证命令
- 完成标准
- 是否需要 plan / todo / subagent

### 执行步骤
1. 读代码与相关上下文
2. 建立 plan / 更新 `TODOS.md`
3. 精确修改目标文件
4. 运行验证命令
5. 检查 diff 与结果
6. 输出变更说明、验证结果、剩余风险

### 完成 Checklist
回复前必须检查：
- 是否改到了目标文件
- 是否运行了验证 / 测试 / 构建
- 是否说明了验证结果
- 是否没有引入新的明显错误
- 是否说明风险、限制、下一步
- 是否有命令输出或结果证据

---

## 4. 工作流类任务模板

### 任务识别
适用于：
- cron / 定时任务
- 多步骤自动化
- 子代理编排
- 文档流转 / 回链 / 交付流程

### 启动 Bootstrap
开始前必须确认：
- 是否为多步骤任务
- 是否需要 subagent
- 最终由谁汇总
- 最终产物落在哪里
- 失败后如何验证和回查

### 执行步骤
1. 定义输入、输出、目标位置
2. 拆分子步骤并写入 `TODOS.md`
3. 配置执行流程或模板
4. 校验每一步产物
5. 汇总结果并返回最终交付物

### 完成 Checklist
回复前必须检查：
- 是否所有子任务都完成
- 是否做了汇总而不是简单拼接
- 是否验证关键结论或关键产物
- 是否产出了最终可交付结果
- 是否落到用户指定位置
- 是否有执行成功的证据

---

## 使用规则

1. 多步骤任务必须先匹配一个任务模板。
2. 匹配到模板后，必须执行对应 bootstrap。
3. 回复前必须执行对应 checklist。
4. 如果任务跨多类，选主模板，再补充次模板约束。
5. 若没有合适模板，先补模板，再执行任务。
